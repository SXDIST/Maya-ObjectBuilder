# Security Policy

## Supported versions

This project is under active development. Security fixes apply to the current `main` branch unless a release branch is explicitly maintained.

## Reporting a vulnerability

Please do not disclose security issues publicly before maintainers have had time to investigate.

Report vulnerabilities by contacting the repository owner through GitHub. Include:

- a concise description of the issue;
- affected files, commands, or workflows;
- reproduction steps where possible;
- any known impact or mitigation.

## Scope

Relevant security issues include unsafe file parsing, unsafe path handling, command execution risks, or vulnerabilities introduced by importer/exporter workflows.

Private DayZ/P3D fixtures, proprietary game assets, credentials, and local machine paths should not be included in public reports.
